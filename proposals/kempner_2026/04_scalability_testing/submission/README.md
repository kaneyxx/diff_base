# Scalability Testing — Submission Package

This folder is the **clean, submission-ready subset** of the Kempner 2026
scalability-testing proposal section. It contains sanitised launch scripts
(personal cluster paths replaced with shell-variable placeholders), the final
result artefacts (CSV summary + PNG plots), and the Technical Readiness
reference template. Everything needed to reproduce the 1/2/4-GPU LoRA sweep
and regenerate the figures is here; raw per-job KempnerPulse CSVs and
per-job SLURM logs live in the parent directory as an audit trail.

For the full development version (raw 1-Hz CSVs, per-job logs, generated
configs), see [../README.md](../README.md).

---

## Reproduce on a 4×A100 Reservation Node

Set three environment variables to match your cluster layout, then submit:

```bash
# 1. Set cluster-specific roots
export REPO_ROOT=/home/you/diff_base          # path to this repository
export MODEL_HUB_ROOT=/home/you/huggingface  # HuggingFace cache root
export DATA_ROOT=/home/you/bao               # BAO/ORION data root

# 2. Make scripts executable
chmod +x slurm_4gpu.sbatch train_wrapper.sh submit_scal_sweep.sh \
         submit_rank_sweep.sh gpu_metrics_recorder.sh

# 3. Submit the default sweep (1 / 2 / 4 GPU, LoRA rank 16)
./submit_scal_sweep.sh

# 4. (Optional) Run the full rank × GPU sweep (rank 8/16/32/64 × 1/2/4 GPU = 12 jobs)
./submit_rank_sweep.sh

# 5. Parse results after jobs complete
python parse_pulse_csv.py results/kempnerpulse_1gpu_<JOBID>.csv 1
python parse_pulse_csv.py results/kempnerpulse_2gpu_<JOBID>.csv 2
python parse_pulse_csv.py results/kempnerpulse_4gpu_<JOBID>.csv 4
```

Expect ~40 min wall-time per job on an A100 80GB node (150 training steps
with gradient checkpointing). Output CSVs land in `results/`; plots are
regenerated by running `python plot_scaling_results.py`.

## File Glossary

| File / Path | Purpose |
|---|---|
| `slurm_4gpu.sbatch` | SLURM job script — sets up environment, starts KempnerPulse recorder, calls `train_wrapper.sh` |
| `train_wrapper.sh` | Inner launcher — fills `scal_lora_template.yaml` from env vars, runs `train.py` via accelerate |
| `scal_lora_template.yaml` | Kontext LoRA config template; `__PLACEHOLDERS__` filled by `train_wrapper.sh` |
| `submit_scal_sweep.sh` | Submit 1/2/4-GPU jobs independently so each starts as GPUs free up |
| `submit_rank_sweep.sh` | Outer loop — iterates over LoRA ranks (8/16/32/64) and calls `submit_scal_sweep.sh` |
| `gpu_metrics_recorder.sh` | nvidia-smi fallback recorder; writes KempnerPulse-aligned CSV when DCGM unavailable |
| `parse_pulse_csv.py` | Reduces a KempnerPulse CSV to a single markdown table row (peak mem, avg util) |
| `plot_scaling_results.py` | Generates the four PNG plots from `results/scaling_summary.csv` |
| `results/summary_table.md` | Final summary table — throughput, memory, and efficiency across GPU counts |
| `results/scaling_summary.csv` | Machine-readable source for the summary table and plots |
| `results/plots/scaling_efficiency.png` | Parallel scaling efficiency (ideal vs measured) |
| `results/plots/gpu_util_timeseries.png` | GPU utilisation over time for 1/2/4-GPU runs |
| `results/plots/memory_profile.png` | Per-GPU memory usage across GPU-count configurations |
| `results/plots/power_profile.png` | Power draw over time for 1/2/4-GPU runs |
| `refs/Technical Readiness Template.docx.pdf` | Kempner Technical Readiness section template |

## Reading the Results

**Summary table** — `results/summary_table.md` lists throughput (steps/s),
peak per-GPU memory (GB), and parallel efficiency for each GPU count. The
efficiency column (1-GPU throughput × N vs N-GPU throughput) is the key
metric for the Kempner Technical Readiness section.

**Plots** — four PNG files in `results/plots/`:

- `scaling_efficiency.png` — bar chart comparing ideal linear scaling vs
  measured scaling efficiency at 1, 2, and 4 GPUs.
- `gpu_util_timeseries.png` — 1-Hz GPU utilisation traces for all three
  configurations overlaid, showing steady-state utilisation during training.
- `memory_profile.png` — per-GPU memory consumption; verifies headroom
  within the 80GB A100 VRAM budget at all GPU counts.
- `power_profile.png` — GPU power draw over time; used to estimate
  energy-per-step in the Kempner sustainability section.

For the dev/raw-data version (individual per-job CSVs, SLURM logs, generated
