This directory contains a frozen copy of the BAO data-loader contract, snapshotted on 2026-05-14 from `/n/scratch/users/f/fas994/bao/src/data/`. The two files — `multi_patient_dataset.py` (`MultiPatientBiomarkerDataset`) and `crc01_dataset.py` — define the H5 tile schema and JSON training-splits format used by the ORION corpus: each H5 file stores tiles as three-channel images where R = biomarker-specific signal, G = Pan-CK (epithelium marker), and B = Hoechst (nuclear stain). Training splits are encoded as JSON with fields `(crc_id, h5_path, he_dir, index, coordinate, sample_type)` for balanced positive/negative tile sampling across patients. These files are included so this proposal remains self-contained and reproducible even if the source scratch directory `/n/scratch/users/f/fas994/bao/` is purged or relocated after the project concludes.
